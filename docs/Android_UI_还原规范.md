# Android UI 还原规范：相册清理工具

**版本：** v0.1  
**用途：** 指导 Android Compose 开发尽量还原当前高保真原型的视觉和组件状态。  
**范围：** V1.0 MVP，高保真 UI 方向稿，不包含 AI、重复对比、社媒预览等后续功能。

---

## 1. 视觉定位

- 风格关键词：清爽、可信、轻量、工具感、低负担。
- 视觉重点：照片内容、当前决策卡片、整理进度、撤回安全感。
- 避免方向：过度游戏化、重装饰、营销感大标题、复杂渐变背景。

## 3. 字号与字重

| 层级 | 建议字号 | 字重 | 用途 |
|---|---:|---:|---|
| PageTitle | 24sp | 800-850 | 首页、设置、统计、精选集标题 |
| SectionTitle | 17sp | 800 | 页面内一级模块标题 |
| CardTitle | 14-15sp | 700-800 | 分组标题、文件名 |
| Body | 13-14sp | 500-650 | 常规正文 |
| Caption | 11-12sp | 500-700 | 日期、数量、说明、状态 |
| NumberHero | 28-34sp | 850-900 | 数据统计和总量 |

Android 可用 `FontWeight.Bold / ExtraBold` 近似；不要求完全等同 Web 字重。

## 6. 图标规范

- 原型使用内置线性 SVG 图标，Android 建议映射到 Material Symbols / Material Icons Outlined。
- 线宽：约 2dp。
- 未选中导航：`subtle`。
- 选中导航：`primary` + `primarySoft` 背景。
- 钉住图标：
  - 未钉住：白色半透明底、`stage` 描边、`stage` 图标线色。
  - 已钉住：`stage` 填充、白色图标线色。

---

## 7. 照片占位与真实图片

当前 Web 原型使用照片式渐变占位图模拟真实照片。Android 实现时：

- 使用真实 MediaStore 缩略图替代占位图。
- 图片必须 `ContentScale.Crop`。
- 缩略图列表加载低清图，决策卡片加载中等清晰度图。
- 视频可在右下角加轻量视频标识，但 MVP 不做视频内容摘要。
- 图片圆角按组件规范裁剪，不额外加厚边框。

## 8. 关键组件状态

### 8.1 底部导航

- 高度：64dp。
- 背景：白色 96% 或纯白。
- 选中项：图标有 `primarySoft` 胶囊底，文字 `primary`，字重 700。
- 未选中项：图标和文字 `subtle`。

### 8.2 首页分组卡片

- 卡片圆角 18dp，白底，1dp `line` 边框，轻阴影。
- 分类图标放在 40dp 圆角底板中。
- 展开后小组行左侧使用真实缩略图。
- 小组行点击进入卡片决策页。

### 8.3 卡片决策页

- 顶部显示小组名、日期和进度百分比。
- 缩略图横幅为等尺寸横向列表，当前项加 `primary` 边框。
- 主照片卡圆角 26dp，阴影较强。
- 右上角钉住按钮固定在照片卡内部。
- 四向提示低调显示，不抢照片主体。
- 底部辅助操作栏默认关闭；设置开启后显示删除、跳过、加入精选集、撤回按钮。

### 8.4 暂存栏

- 位于卡片页底部，白色卡片，圆角 18dp。
- 折叠状态显示标题和数量。
- 展开后显示横向缩略图。
- 进入暂存页后只做人工筛选，不做放大浏览和预览。

### 8.5 精选集托盘

- BottomSheet，顶部圆角 24dp。
- 顶部有 42dp 宽拖拽条。
- 标题：加入精选集。
- 副文案：仅在本应用内整理。
- 图标使用线性图标，不使用 emoji。

### 8.6 Toast

- 黑色 90% 背景，白字，胶囊圆角。
- 位置：卡片页底部，避开辅助操作栏。
- 文案：
  - `已撤回删除`
  - `已撤回跳过`
  - `已撤回暂存`
  - `没有可撤回的操作`

## 9. 第一批最终风格页面

第一批锁定以下页面作为 Android 开发视觉基准：

- 首页分组列表
- 卡片决策页
- 设置页

这些页面的颜色、间距、圆角、层级应优先还原。精选集页、统计页、暂存页作为第二批微调对象。
