## 一、产品背景

### 1.1 问题陈述

智能手机用户每天产生大量照片、截图与下载图片，相册随时间推移迅速膨胀，导致：

- **存储空间紧张**：大量重复、低质量、无用图片占据宝贵空间；
- **整理效率低下**：传统相册按时间线铺排，缺乏语义分组，用户难以快速找到"值得保留"的内容；
- **清理心理负担重**：逐张判断删除与否，操作繁琐、容易疲劳放弃；
- **收藏管理混乱**：系统默认收藏夹单一，难以按主题归档。

### 1.2 产品定位

一款以"**轻松决策、快速行动**"为核心设计理念的手机相册清理与归档工具。MVP 阶段优先通过基础分组和直觉手势降低整理门槛，让用户像刷卡片一样快速完成相册清理；AI 智能分组作为后续增强能力迭代。

## 二、目标用户

### 2.1 核心用户画像

| 用户群体 | 特征描述 | 核心痛点 |
|---|---|---|
| 学生群体（18-25岁） | 拍摄毕业照、聚会、旅行；截图量大 | 相册杂乱，找不到重要纪念照 |
| 职场人士（25-35岁） | 工作截图、文档扫描、生活记录混杂 | 工作与生活内容混在一起，清理无从下手 |
| 家庭用户（30-45岁） | 孩子成长照大量累积，存储告急 | 存储空间不足，不知道哪些可以删 |

### 2.2 用户目标

- **首要目标**：释放手机存储空间；
- **次要目标**：整理有纪念价值的照片便于回顾；
- **体验目标**：整理过程轻松愉快，有成就感。

## 三、产品目标与成功指标

### 3.1 产品目标

1. 用户单次整理会话平均处理照片数 **≥ 50 张**；
2. 用户 7 日留存率 **≥ 40%**；
3. 用户平均每次使用释放存储空间 **≥ 200MB**；
4. 核心手势操作学习成本 **≤ 30 秒**（新用户引导后）。

### 3.2 成功指标（KPI）

| 指标 | 目标值 |
|---|---|
| 次日留存率 | ≥ 55% |
| 单次会话时长 | 5~15 分钟 |
| 误操作撤回率 | ≤ 10% |
| 基础分组可解释性 | 用户可理解当前小组来源 |
| 崩溃率 | ≤ 0.1% |

## 四、核心功能规格

### 4.1 分组功能

#### 4.1.1 分组维度

系统按版本逐步支持以下分组策略。MVP 阶段仅提供基础分组，AI 事件识别与语义分类进入 V1.5 及后续版本。

```
基础分组（MVP）
    ├── 按时间分组：同日期 / 相邻时间段
    ├── 按文件类型分组：照片 / 视频 / 截图
    └── 按来源分组：相机拍摄 / 截图 / 下载图片

事件识别分组（V1.5）
    └── 拍摄类（相机拍摄）
        ├── 事件聚合：同日期 + 同场景 + 同人物 → 一个小组
        │   示例："2026-06-20 毕业典礼" / "2026-06-22 毕业照"
        └── 大分类归并：相同主题的跨日期小组归入大分类
            示例：[毕业照] 大分类 → 包含多个日期小组
    └── 截图类（来源应用分类）
        ├── 微信截图
        ├── 微博截图
        ├── 浏览器截图
        └── 其他 App 截图
    └── 下载类（内容语义分类）
        ├── 风景 / 建筑
        ├── 人物 / 肖像
        ├── 食物
        ├── 文字 / 文档
        └── 表情包 / 趣图
```

#### 4.1.2 分组层级结构

```
大分类（Big Group）
  └── 小组（Group）
        └── 单张照片（Photo Item）
```

- **大分类**：语义主题层（如"毕业照"、"旅行"、"工作截图"）
- **小组**：时间 + 场景聚合层（如"2026-06-20 毕业典礼"）
- 用户在"小组"层级进行批量操作，降低决策粒度

#### 4.1.3 分组智能识别规则

| 图片类型 | 识别策略 | 分组依据 |
|---|---|---|
| 相机拍摄 | MVP：按拍摄日期 / 时间段聚合；V1.5：增加场景/人脸聚类 | MVP 不依赖 AI 模型 |
| 截图 | MVP：按系统媒体类型或路径特征识别；V1.5：增加来源 App 识别 | 优先保证可解释和速度 |
| 下载图片 | MVP：按目录 / 修改时间聚合；V1.5：增加图像语义分类 | MVP 不做内容语义判断 |
| 重复图片 | V1.5：感知哈希（pHash）相似度检测 | MVP 暂不支持自动重复检测 |
| 视频文件 | MVP：按时间 / 文件类型分组 | 与图片共用处理流程，但不做视频内容分析 |

#### 4.1.4 分组 UI 设计要点

- 分组选择控件（Checkbox / 圆形选中按钮）位于**卡片右上角**，不遮挡主体内容；
- 支持**一键全选当前小组**；
- 选中状态视觉反馈明显（边框高亮 + 角标数量显示）；
- 分组折叠/展开动画流畅；
- 进入某小组处理时，顶部展示**轻量缩略图横幅栏**：
  - MVP：横向缩略图列表 + 当前项高亮 + 点击跳转；
  - V1.5：评估加入中央焦点 / 景深动画；
  - 低端设备优先保证滚动和手势帧率，不强依赖复杂动效。

### 4.2 手势操作功能

#### 4.2.1 手势映射定义

在**单张照片查看/决策模式**下，手势操作如下：

| 手势 | 动作 | 视觉反馈 | 说明 |
|---|---|---|---|
| **上划** | 删除 | 卡片上飞出 + 红色遮罩 + 震动反馈 | 直接调用 Android MediaStore API 移入系统回收站；首次删除前展示轻提示 |
| **下划** | 加入精选集 | 卡片下坠 + 金色/黄色遮罩 | 触发整理夹选择浮层（见 4.3） |
| **左滑** | Pass（跳过） | 卡片向左飞出 + 灰色遮罩 | 标记为"本次跳过"，不做任何处理 |
| **右滑** | 撤回上一步 | 上一张照片从右侧滑回 + 绿色闪烁 | 可撤回删除、加入精选集、Pass、暂存四种操作 |
| **点击右上角钉住图标** | 暂存（辅助操作） | 图标由描边变为填色 + 暂存栏数量角标 +1 | 将照片加入暂存栏，供后续批量处理 |

> **说明：** 暂存操作为点击触发而非手势，因其核心场景是"先放一放、回头再说"，无需快速连续操作。钉住图标固定在卡片右上角，未钉住时仅显示描边，钉住后显示填色状态。

#### 4.2.2 辅助操作栏

- 底部辅助操作栏默认关闭，避免干扰核心手势学习；
- 用户可在设置页打开"显示辅助操作栏"；
- 开启后底部展示删除、跳过、加入精选集、撤回等按钮，作为手势之外的可访问性入口；
- 暂存不放入底部辅助操作栏，始终通过卡片右上角钉住图标操作。

#### 4.2.3 手势识别参数

- 触发手势的最小滑动距离：**60dp**；
- 手势方向判定容差：**±30°**；
- 手势触发后提供 **500ms 动画**过渡，增强操作实感；
- 支持**长按预览**照片详情（EXIF 信息、文件大小、拍摄设备）。

#### 4.2.4 操作反悔机制

- 右滑撤回支持连续撤回，最多可撤回**当前小组内的所有操作**；
- 首次执行删除操作前展示一次轻提示，说明照片将移入 Android 系统回收站，可通过右滑撤回；
- 轻提示底部提供"不再提醒"勾选框；用户确认后，本设备后续删除不再弹窗打扰；
- 除首次删除轻提示外，其余操作均直接实时执行，不设置二次确认弹窗；
- 撤回删除操作时，调用 Android 系统回收站还原接口将文件恢复至原始位置。

### 4.3 精选集 / 整理夹管理功能

#### 4.3.1 定位说明

- 本功能命名为"精选集"或"整理夹"，避免与 Android / 系统相册的"收藏"概念混淆；
- 精选集是**应用内独立整理体系**，用于帮助用户按主题保存值得保留的媒体引用；
- 加入精选集不会同步到系统相册收藏，也不会改变系统相册中的收藏状态；
- 页面与提示文案需明确说明："精选集仅在本应用内管理，可按需导出到系统相册"。

#### 4.3.2 下划触发加入精选集流程

```
用户下划照片
    → 照片进入拖拽悬停状态（半透明浮在手指下方）
    → 底部托盘从屏幕底部上滑浮现（动画时长 200ms）
    → 托盘展示多个精选集/整理夹图标（横向排列）
    → 用户拖拽至目标精选集图标上方松手
    → 加入成功动画（图标弹跳 + 数量+1）
    → 托盘自动收起，进入下一张照片
```

#### 4.3.3 精选集托盘设计

- 托盘高度：**80dp**，圆角浮层样式，背景半透明毛玻璃效果；
- 默认预设精选集（可自定义名称和图标）：

| 图标 | 名称 | 用途说明 |
|---|---|---|
| ⭐ | 精选 | 最值得保留的照片 |
| ✈️ | 旅行 | 出行相关 |
| 👨‍👩‍👧 | 家人 | 家庭成员 |
| 💼 | 工作 | 工作相关文档/截图 |
| ➕ | 新建 | 创建新精选集 |

- 最多同时展示 **5个精选集 + 1个新建入口**；
- 精选集支持自定义：名称、Emoji图标、封面照片；
- 拖拽至"新建"时，弹出快速命名输入框，创建后立即加入精选集。

#### 4.3.4 精选集主页面

- **应用内独立精选集体系**，与 Android 系统相册原生收藏完全隔离，不读取也不写入系统收藏；
- 精选集数据以应用私有数据库存储（仅保存媒体文件引用路径，不复制原文件）；
- 独立的精选集管理页，展示所有精选集及其封面；
- 支持精选集内部的二次整理（排序、移动、删除照片）；
- 支持将精选集导出为系统相册或分享。

#### 4.4.2 数据统计功能

统计面板聚焦清理成果展示，保持轻量简洁：

| 统计维度 | 内容 |
|---|---|

### 4.5 重复图片对比功能

> 版本定位：重复图片检测与两两对比不进入 MVP，规划为 V1.5 增强能力。MVP 仅保留手动暂存和人工筛选，避免首版引入图像哈希计算、对比交互和误删风险。

#### 4.5.1 重复识别与标记（V1.5）

- V1.5 在分组扫描阶段自动检测相似/重复图片（感知哈希 pHash，相似度 ≥ 90%）；
- 相似重复的图片/视频在缩略图右上角自动显示 📌 标识符；
- 用户可手动为任意照片/视频添加或移除 📌 标记，以备后续对比使用。

#### 4.5.2 两两对比模式（V1.5）

**触发方式：** 点击任意带有 📌 标记的媒体文件进入对比选择界面；选择第二张 📌 媒体文件后，进入并排对比界面。

**对比界面功能：**

| 功能 | 说明 |
|---|---|
| 并排显示 | 两张图片左右并排等比展示 |
| 独立缩放 | 双指捕合对任意一侧单独放大，另一侧保持不动 |
| 联动模式 | 开启后两侧同步缩放/平移，便于像素级细节对比 |
| 信息对比栏 | 底部显示两张图片的文件大小、分辨率、拍摄时间 |
| 一键保留 | 点击“保留此张”，另一张自动移入 Android 系统回收站 |

## 五、交互设计要点

### 5.1 核心交互流程

```
启动 App
  → 扫描相册（MVP：读取媒体与基础分组）
  → 分组结果展示页（大分类 + 小组列表）
  → 用户选择一个小组进入
  → 卡片滑动决策模式（手势操作，操作实时生效）
  → 小组处理完成（庆祝动效 + 进度更新）
  → 返回分组列表选择下一组
  → 所有分组处理完毕 → 完成页（成就展示）
```

**实时操作模式说明：**
- 除首次删除轻提示外，删除、加入精选集、跳过等操作均实时执行，无需手动保存；
- 应用退出后不保留分组进度状态；
- 下次进入应用时，系统对设备当前相册中的现有媒体文件**重新扫描分组**，以最新相册状态为准。

### 5.2 新用户引导设计

- 首次启动提供 **交互式手势教学**（非视频，实际操作练习）；
- 引导照片使用虚拟示例，不读取真实相册；
- 引导步骤 ≤ 4步，每步单一手势，30秒内完成；
- 引导完成后给予"整理达人"入门徽章激励。

### 5.3 操作反馈设计

| 反馈类型 | 场景 | 设计要求 |
|---|---|---|
| 触觉反馈 | 手势触发成功 | 短促震动（HapticFeedback） |
| 视觉反馈 | 卡片飞出动画 | 60fps 流畅动画，符合手势方向物理感 |
| 声音反馈（可选） | 删除/加入精选集成功 | 轻柔音效，默认关闭，用户可开启 |
| 成就反馈 | 完成一个小组 | 轻量庆祝动效（粒子/星星爆破） |
| 空状态 | 相册已全部整理 | 插画 + 激励文案，引导分享战报 |

### 5.4 防误操作设计

- 首次删除前展示一次轻提示，支持勾选"不再提醒"；确认后删除直接实时执行，通过右滑手势撤回误操作；
- 删除操作通过 Android 系统 MediaStore 回收站机制实现，照片可在系统相册回收站中恢复；
- 右滑撤回删除时，调用系统回收站还原接口，照片回到原始位置；
- 应用退出后不保留进度，再次进入时重新扫描分组。

## 六、技术实现建议

### 6.1 MVP 技术架构

| 模块 | 建议方案 | 说明 |
|---|---|---|
| 媒体读取 | Android MediaStore | 读取照片 / 视频基础元数据、缩略图 URI、文件大小 |
| 删除与恢复 | MediaStore Trash API | Android 11+ 使用系统回收站；Android 10 需做兼容策略 |
| 本地存储 | Room / SQLite | 保存精选集、操作撤回栈、用户设置 |
| 图片加载 | Coil / Glide | 缩略图缓存、列表滚动、卡片预览 |
| 手势交互 | Jetpack Compose pointerInput 或 View DragHelper | 实现四向手势、阈值、动画和触觉反馈 |
| 权限 | Android Photo / Media 权限 | 按系统版本申请最小必要权限 |

**隐私原则：MVP 不申请网络权限，不上传图片数据；所有整理数据保存在设备本地。**

### 6.2 后续端侧 AI 能力

| 能力模块 | 建议方案 | 版本 | 说明 |
|---|---|---|---|
| 场景/事件识别 | TensorFlow Lite | V1.5 | 本地推理，保护隐私 |
| 人脸聚类 | ML Kit (Android) | V1.5 | 同场景同人物聚合 |
| 图像分类（截图/下载） | MobileNetV3 轻量化模型 | V1.5 | 模型随安装包内置，不做首次下载 |
| 重复图片检测 | 感知哈希（pHash） | V1.5 | 速度快，准确率高 |
| OCR（截图文字） | 系统原生 OCR API | V2.0 | 避免额外依赖 |

### 6.3 性能要求

| 场景 | 性能目标 |
|---|---|
| MVP 相册扫描（1000张） | ≤ 10秒完成基础分组 |
| V1.5 AI 分析（1000张） | ≤ 30秒完成分析 |
| 手势动画帧率 | 稳定 60fps |

### 7.1 V1.0 MVP 必做清单

- 基础分组：按日期、文件类型、来源目录/媒体类型生成大分类和小组；
- 图片与视频统一展示，但视频不做内容分析；
- 四向手势：删除、加入精选集、跳过、撤回；
- 底部辅助操作栏默认关闭，可在设置中打开；
- 首次删除轻提示，支持"不再提醒"，删除进入 Android 系统回收站；
- 应用内精选集：默认精选集、旅行、家人、工作，不同步系统收藏；
- 暂存栏：右上角钉住、移除、进入暂存页人工筛选；
- 轻量缩略图横幅：当前项高亮、左右滚动、点击跳转；
- 统计面板：本次/累计删除数量、释放空间；
- 浅色 / 深色主题；
- 完全本地运行，不申请网络权限。

### 7.2 V1.0 明确不做

- 不做 AI 事件识别、人脸聚类、图像语义分类；
- 不做重复图片自动检测和两两对比；
- 不做 OCR、照片压缩、云端备份联动；
- 不做系统相册收藏同步；
- 不做复杂景深缩略图动画；
- 不做暂存图片放大浏览、发布预览或社媒预览（注：`PublishPreviewScreen` 代码已实现但入口隐藏作为 V1.5 储备，详见第十章 Open Issues #8）；
- 不做分享战报。

## 八、开发落地方案

### 8.1 推荐技术栈

| 层级 | 建议 | 说明 |
|---|---|---|
| UI | Jetpack Compose | 手势、动画、状态驱动界面更直接 |
| 架构 | MVVM + Repository | ViewModel 管理会话状态，Repository 封装媒体和数据库 |
| 本地数据库 | Room | 保存精选集、精选集条目、用户设置、撤回动作 |
| 媒体访问 | MediaStore | 查询媒体、缩略图、文件大小、删除到回收站 |
| 图片加载 | Coil | Compose 适配好，支持缩略图缓存 |
| 后台任务 | Kotlin Coroutines / WorkManager | 扫描、统计、后续 AI 分析任务调度 |

### 8.2 核心数据结构

| 数据对象 | 关键字段 | 用途 |
|---|---|---|
| PhotoItem | mediaId、uri、type、dateTaken、sizeBytes、duration、sourceBucket | 媒体基础展示与操作 |
| MediaGroup | groupId、title、groupType、photoIds、coverUri、count、totalSize | 分组列表和小组入口 |
| Collection | collectionId、name、icon、coverUri、createdAt | 应用内精选集 |
| CollectionItem | collectionId、mediaId、addedAt、note | 精选集与媒体的引用关系 |
| ActionRecord | actionId、mediaId、actionType、payload、createdAt、groupId | 当前小组内撤回栈 |
| UserSettings | hasSeenDeleteTip、themeMode、hapticEnabled、soundEnabled、actionBarEnabled | 用户偏好 |

### 8.3 开发任务拆解

| 优先级 | 任务 | 交付标准 |
|---|---|---|
| P0 | 媒体权限与 MediaStore 扫描 | 可读取图片/视频，生成基础列表 |
| P0 | 基础分组引擎 | 可按日期、类型、来源生成小组 |
| P0 | 卡片决策页 | 四向手势、动画、触觉反馈可用 |
| P0 | 删除到系统回收站 | 首次提示、删除、撤回恢复链路可用 |
| P0 | 精选集数据库 | 默认精选集、加入、移除、列表展示 |
| P0 | 操作撤回栈 | 删除、加入精选集、跳过、暂存均可撤回 |
| P0 | 暂存栏 | 右上角钉住、展开、移除、进入暂存页 |
| P1 | 缩略图横幅 | 当前项高亮、横向滚动、点击跳转 |
| P1 | 统计面板 | 本次/累计数量和释放空间计算 |
| P1 | 设置页 | 主题、触觉反馈、删除提示、辅助操作栏开关 |
| P2 | 性能优化 | 1000 张基础扫描 ≤ 10 秒，滑动稳定 60fps |

### 8.4 关键风险与边界

| 风险 | 处理策略 |
|---|---|
| Android 10/11+ 回收站 API 差异 | 按系统版本封装 DeleteRepository，低版本给出兼容提示 |
| 用户拒绝媒体权限 | 提供空状态和重新授权入口，不进入主流程 |
| 撤回恢复失败 | 明确提示失败原因，并引导用户到系统回收站恢复 |
| 大相册扫描慢 | 分页加载、先展示基础分组，再后台补充统计 |
| 视频缩略图占用内存 | 使用图片库缓存策略，列表只加载低清缩略图 |
| 精选集被误解为系统收藏 | 页面固定提示"仅在本应用内整理"，导出单独作为动作 |
| 应用退出不保留进度 | 仅保存已执行结果和精选集，不保存未完成队列 |
